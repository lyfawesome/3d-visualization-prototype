# 三维试件模型可视化原型开发计划

## 1. 项目目标

开发一个 Windows 桌面演示原型，用于支撑响应文件中关于三维试件模型可视化的技术承诺：

- 软件可加载专业 3D 软件生成的试件模型文件，自动生成三维可视化场景。
- 软件可在模型表面标注和显示传感器的位置。
- 软件可将实时采集或模拟的载荷测试数据进行色彩映射，实现试件表面载荷分布可视化。

当前阶段以标书响应可信和截图展示为优先目标，不实现完整工业现场联调。

## 2. 交付范围

本项目第一阶段交付一个可运行的 Windows 桌面原型，以及可放入响应文件的功能截图。

核心交付物：

- Windows 桌面程序原型。
- 可导入的样例试件模型。
- 传感器点位配置样例。
- 模拟载荷测试数据样例。
- 三张响应文件截图：
  - 已加载专业 3D 模型并生成三维可视化场景。
  - 模型表面显示传感器位置和编号。
  - 载荷数据驱动的表面色彩云图。

不在第一阶段范围内：

- 真实 PLC、DAQ 或控制系统实时联调。
- 工程级有限元求解或仿真精度验证。
- 用户权限、审计日志、长期数据库存储等完整产品能力。
- 控制算法、驱动系统或电动缸闭环控制开发。

## 3. 技术路线

建议采用 Qt + VTK + OCCT 技术路线：

- Qt：构建 Windows 桌面应用界面。
- VTK：负责三维渲染、场景交互、传感器标注、颜色映射和截图导出。
- OCCT：负责读取 STEP/STP 等 CAD 工程模型，并转换为可渲染网格。

优先支持模型格式：

- STEP/STP：通过 OCCT 读取。
- STL：通过 VTK 读取。
- OBJ：通过 VTK 读取。

如开发周期受限，可先完成 STL/OBJ 演示链路，再补 STEP/STP 导入能力。

## 4. 功能设计

### 4.1 模型加载与三维场景

用户可在界面中选择模型文件。软件读取模型后自动完成：

- 模型导入。
- 模型居中。
- 自动缩放到合适视野。
- 创建三维相机、光照、坐标轴和背景。
- 支持鼠标旋转、滚轮缩放、平移和视角重置。

模型加载失败时，应提示格式不支持、文件损坏或导入异常。

### 4.2 传感器标注

软件读取传感器配置文件，将传感器显示在模型表面或模型附近。

建议传感器配置使用 `sensors.json`：

```json
[
  {
    "id": "S01",
    "name": "Load Sensor 01",
    "position": [120.0, 35.0, 8.0],
    "type": "load",
    "unit": "kN"
  }
]
```

界面显示要求：

- 传感器点位使用醒目的点、球体或图标显示。
- 点位旁显示传感器编号。
- 选中传感器时显示名称、坐标、类型、单位和当前值。
- 视角旋转后标注仍应跟随三维位置显示。

### 4.3 模拟载荷数据

第一阶段使用模拟数据，不对接真实采集系统。

建议模拟数据使用 `load_data.csv`：

```csv
timestamp,S01,S02,S03,S04
0.00,12.3,10.8,9.5,13.1
0.10,12.8,11.2,9.8,13.4
```

软件应支持：

- 加载模拟数据文件。
- 按时间步播放数据。
- 暂停、继续、重置播放。
- 显示当前时间和各传感器当前值。

### 4.4 载荷云图可视化

将传感器点值映射到模型表面，形成演示级连续色彩云图。

建议算法：

- 使用反距离权重插值或径向基插值。
- 每个模型表面顶点根据附近传感器值计算颜色。
- 使用蓝到红的色带表示载荷低到高。
- 显示色标图例、最小值、最大值和单位。

该云图用于功能演示和响应文件截图，不声明为工程仿真结果。

## 5. 界面结构

建议主界面包含：

- 顶部工具栏：
  - 打开模型。
  - 加载传感器配置。
  - 加载模拟数据。
  - 播放/暂停。
  - 重置视角。
  - 导出截图。
- 左侧面板：
  - 当前模型信息。
  - 传感器列表。
  - 当前数据时间步。
- 中央区域：
  - 三维模型视窗。
  - 传感器标注。
  - 云图和色标。
- 底部状态栏：
  - 文件加载状态。
  - 当前帧率或渲染状态。
  - 错误提示。

## 6. 截图要求

响应文件至少准备三张截图。

截图一：模型加载场景

- 显示完整试件模型。
- 显示三维坐标轴或方向辅助。
- 模型处于清晰可辨识角度。
- 文件名或模型信息可见。

截图二：传感器标注

- 显示多个传感器点位。
- 显示传感器编号。
- 侧边栏展示传感器列表或选中传感器信息。

截图三：载荷云图

- 模型表面显示连续色彩映射。
- 显示色标图例。
- 显示当前时间步或模拟实时数据状态。

## 7. 验收标准

第一阶段完成后应满足：

- 可在 Windows 上启动桌面程序。
- 可成功导入至少一个样例 STL/OBJ 模型。
- 可成功导入或展示一个 STEP/STP 模型，或提供明确的 STEP 转换链路。
- 可显示三维场景并进行缩放、旋转、平移。
- 可加载传感器配置并显示传感器点位和编号。
- 可加载模拟载荷数据并随时间刷新传感器值。
- 可生成模型表面载荷云图。
- 可导出用于响应文件的 PNG 截图。

## 8. 测试计划

### 8.1 模型导入测试

- 测试 STL 文件能正常加载。
- 测试 OBJ 文件能正常加载。
- 测试 STEP/STP 文件能正常加载或转换。
- 测试文件不存在、格式错误、空模型时的错误提示。

### 8.2 场景交互测试

- 测试鼠标旋转。
- 测试滚轮缩放。
- 测试平移。
- 测试视角重置。
- 测试窗口缩放后模型仍正常显示。

### 8.3 传感器标注测试

- 测试传感器配置文件读取。
- 测试点位显示位置。
- 测试编号显示。
- 测试传感器选中和详情显示。

### 8.4 数据播放测试

- 测试模拟数据加载。
- 测试播放、暂停、重置。
- 测试当前时间步显示。
- 测试传感器数值随时间变化。

### 8.5 云图测试

- 测试颜色随载荷值变化。
- 测试色标最小值、最大值和单位显示。
- 测试极值、缺失值和异常值处理。
- 测试截图导出结果清晰可用。

## 9. 后续扩展

现场可用版本可在原型基础上扩展：

- 对接真实 DAQ/PLC/上位机数据接口。
- 增加模型坐标系与传感器安装坐标校准工具。
- 增加传感器配置编辑和保存功能。
- 增加数据记录、回放和报告导出。
- 增加异常报警、阈值配置和通道状态显示。
- 增加工况管理和多模型管理。

## 10. 默认假设

- 当前阶段只做演示原型。
- 样例模型由项目准备，不依赖客户真实模型。
- 载荷数据使用模拟数据。
- 云图为演示级插值，不作为工程仿真精度证明。
- 响应文件截图优先展示功能完整性和视觉可信度。
