# 3D Visualization Prototype

This repository contains a Windows desktop prototype for 3D specimen model visualization.

## Goal

The prototype demonstrates these functions:

- Load specimen model files generated by professional 3D/CAD software.
- Automatically create an interactive 3D visualization scene.
- Mark and display sensor positions on the specimen model.
- Map simulated load data to the model surface as a color contour visualization.

## Technical Stack

- C++17
- Qt 6 Widgets for the Windows desktop interface
- VTK for 3D rendering, interaction, labels, color mapping, and screenshot export
- OpenCASCADE / OCCT for direct STEP/STP CAD import
- CMake and MSYS2 UCRT64 for Windows dependency deployment with prebuilt binary packages

Supported model formats:

- STEP/STP, read through OCCT `STEPControl_Reader`
- STL, read through VTK
- OBJ, read through VTK

## Repository Contents

- `src/`: C++ Qt/VTK/OCCT prototype source code.
- `samples/specimen.stl`: simple sample specimen model.
- `samples/sensors.json`: sample sensor positions.
- `samples/load_data.csv`: sample simulated load data.
- `CMakeLists.txt`: CMake project definition.
- `CMakePresets.json`: MSYS2 UCRT64 + Ninja preset.
- `scripts/build_windows.ps1`: Windows configure/build script.
- `scripts/run_windows.ps1`: build-if-needed and run script.
- `3d-visualization-plan.md`: detailed development target, feature scope, interface assumptions, screenshots, and test plan.

## Build on Windows

Prerequisites:

- Windows 10/11
- MSYS2 installed at `C:\msys64`
- Git
- PowerShell

Run:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\scripts\build_windows.ps1
```

The script installs the required MSYS2 UCRT64 binary packages if they are missing, then configures CMake and builds the Release executable with Ninja.

Core MSYS2 packages:

- `mingw-w64-ucrt-x86_64-qt6-base`
- `mingw-w64-ucrt-x86_64-vtk`
- `mingw-w64-ucrt-x86_64-opencascade`
- `mingw-w64-ucrt-x86_64-cmake`
- `mingw-w64-ucrt-x86_64-ninja`
- `mingw-w64-ucrt-x86_64-gcc`

Run the prototype:

```powershell
.\scripts\run_windows.ps1
```

## Current Prototype Features

- Loads the included STL sample automatically at startup.
- Supports manual STEP/STP, STL, and OBJ model loading.
- Converts OCCT STEP topology into `vtkPolyData` in memory.
- Shows sensor points and 3D labels from `sensors.json`.
- Plays simulated load frames from `load_data.csv`.
- Maps sensor load values onto model surface points with inverse-distance weighting.
- Exports the render viewport to PNG.
